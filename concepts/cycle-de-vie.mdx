## Objectif
 
Cette page décrit les grandes phases que traverse toute intervention sur Intervimo. Elle sert de carte d'orientation : avant de plonger dans votre guide dédié, comprenez ici ce qui se passe à chaque étape et qui en est responsable.
 
## Vue d'ensemble
 
```
Signalement → Qualification → Matchmaking → Devis → Exécution → Archive
```
 
Chaque intervention naît d'un **signalement** et suit ce chemin dans l'ordre. Certaines phases peuvent être plus courtes (le devis est parfois facultatif), mais la séquence reste la même.

 
## Les 6 phases
 
### 1. Signalement
 
**Qui agit :** le locataire ou propriétaire (résident), ou le gestionnaire.
 
Un problème est constaté dans un logement. Le résident le déclare depuis son interface ou le gestionnaire le crée directement depuis son cockpit. L'intervention est créée avec la phase **CREATED** et une fiche de qualification est générée automatiquement.

### 2. Qualification
 
**Qui agit :** le gestionnaire.
 
Le gestionnaire examine le signalement. Il peut demander des précisions au résident si la description est insuffisante, puis approuver ou rejeter l'intervention. Cette phase permet de valider que la demande est légitime et suffisamment documentée avant d'engager un prestataire.
 
→ Si approuvée, l'intervention passe en **Matchmaking**. Si rejetée, elle est clôturée avec un motif.
 
### 3. Matchmaking
 
**Qui agit :** le gestionnaire, puis le prestataire.
 
Le gestionnaire identifie un ou plusieurs prestataires adaptés au type d'intervention et envoie un **mandat**. Le prestataire dispose de 48 heures pour accepter ou refuser. En cas de refus ou d'absence de réponse, un prestataire de remplacement peut être sollicité.
 
→ Dès qu'un mandat est accepté, l'intervention passe en phase **Exécution**.
 
### 4. Devis
 
**Qui agit :** le prestataire, puis le gestionnaire.
 
Cette phase n'est pas systématique. Si l'intervention nécessite un chiffrage préalable, le prestataire soumet un **devis** que le gestionnaire peut approuver, refuser ou renvoyer en révision. Sans devis requis, cette phase est ignorée et l'exécution démarre directement.

 
### 5. Exécution
 
**Qui agit :** le gestionnaire, le prestataire et le résident.
 
C'est la phase opérationnelle. Elle couvre la planification du rendez-vous (proposition de créneaux, confirmation), le démarrage des travaux sur site, leur suivi et leur clôture. Le gestionnaire supervise l'avancement depuis son cockpit. Une fois les travaux terminés et validés, le gestionnaire approuve la clôture.
 
### 6. Archive
 
**Qui agit :** le système (automatique après approbation).
 
L'intervention est archivée : elle disparaît de la liste active mais reste consultable dans l'historique. Aucune modification n'est possible sur une intervention archivée.

 
## Ce que cette page ne couvre pas
 
Chaque phase est détaillée dans le guide de l'acteur concerné :
 
- **Gestionnaire** → Guide gestionnaire 
- **Prestataire** → Guide prestataire
- **Résident** → Guide résident
Consultez le [Glossaire](/concepts/glossaire) pour la définition précise de chaque terme.
